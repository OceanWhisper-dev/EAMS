## 2. 部署环境要求

### 2.1 硬件要求

| 环境 | 建议配置 |
|------|---------|
| 开发 | 8GB+ RAM, SSD, 4 核 CPU |
| 生产 | 16GB+ RAM, SSD, 8 核 CPU |

### 2.2 软件要求

| 软件 | 版本 | 说明 |
|------|------|------|
| Windows Server / Linux | - | 生产服务器 OS |
| .NET 8 SDK | 8.0+ | 后端运行时 |
| PostgreSQL | 14+ | 主数据库 |
| Node.js | 18+ | 前端构建（仅构建时需要） |
| SQL Server 2005+ | - | 仅 HWATT 考勤数据同步需要 |

| 端口 | 用途 |
|------|------|
| 5106 (HTTP) | 后端 API |
| 5107 (HTTP) | 前端静态资源 |
| 5432 | PostgreSQL |
| 1433 | SQL Server (ERP/HWATT) |

## 4. 构建与发布

### 4.1 前端构建

```bash
cd src/EAMS2026.Web

# 安装依赖（初次或依赖变更后）
npm install

# 构建生产版本
npm run build    # 输出到 dist/

# 预览构建结果
npm run preview  # 本地预览 :4173
```

### 4.2 后端发布

```bash
cd src/EAMS2026.Api

# 发布到指定目录
dotnet publish -c Release -o ../../publish/api
```

或发布到操作系统特定目录：

```bash
# Windows
dotnet publish -c Release -o ../../publish/api -r win-x64 --self-contained false

# Linux
dotnet publish -c Release -o ../../publish/api -r linux-x64 --self-contained true
```

### 4.3 发布目录结构

```
publish/
├── api/                    # 后端发布文件
│   ├── EAMS2026.Api.dll
│   ├── appsettings.json
│   ├── appsettings.Development.json
│   └── ...
├── web/                    # 前端构建文件
│   ├── index.html
│   ├── assets/
│   └── ...
└── database/               # SQL 脚本
    └── *.sql
```

## 5. 数据库部署

### 5.1 创建数据库

```sql
CREATE DATABASE eams2026;
```

### 5.2 执行建表脚本

按顺序执行 `publish/database/` 目录下的 SQL 脚本：

```bash
# 方法一：psql 命令行
psql -h localhost -U postgres -d eams2026 -f 001_Init.sql

# 方法二：使用 pgAdmin 查询工具逐文件执行
```

> 注意：脚本须按编号顺序执行，部分脚本依赖前置脚本的结果。

### 5.3 连接字符串配置

编辑 `appsettings.json` 中的 `ConnectionStrings:DefaultConnection`：

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=localhost;Port=5432;Database=eams2026;Username=postgres;Password=your_password;Pooling=true;Maximum Pool Size=100"
  },
  "JwtSettings": {
    "SecretKey": "your-32-char-min-secret-key-here!!!!!",
    "Issuer": "EAMS2026",
    "Audience": "EAMS2026",
    "ExpireMinutes": 1440
  }
}
```

## 6. 启动服务

### 6.1 后端启动

```bash
# 开发模式（含 Swagger）
cd src/EAMS2026.Api
dotnet run

# 生产模式
cd publish/api
dotnet EAMS2026.Api.dll

# 指定 URL
dotnet EAMS2026.Api.dll --urls "http://0.0.0.0:5106"
```

### 6.2 前端启动

```bash
# 开发模式（Vite 代理 → :5106）
cd src/EAMS2026.Web
npm run dev

# 生产模式（Nginx 或 IIS 托管静态文件）
# 将 dist/ 目录部署到 Web 服务器
```

### 6.3 Nginx 配置示例

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /path/to/publish/web;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://localhost:5106;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # SignalR WebSocket 支持
    location /hubs/ {
        proxy_pass http://localhost:5106;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
```

## 7. 环境配置差异

### 7.1 开发环境

| 配置 | 值 |
|------|-----|
| 后端 URL | `http://localhost:5106` |
| 前端 URL | `http://localhost:5107`（Vite 代理 → :5106） |
| Swagger | 启用 |
| CORS | 允许所有来源 |

### 7.2 生产环境

| 配置 | 值 |
|------|-----|
| 后端 URL | `http://localhost:5106`（可自定义） |
| 前端 URL | Nginx 代理统一入口 |
| Swagger | 禁用 |
| CORS | 仅允许前端域名 |
| JWT 密钥 | 必须设置复杂字符串 |
