# 部署指南 - EAMS2026 v1.0.20260605

## 1. 部署概述

### 1.1 架构说明

```
                    lxc132 (NGINX 1.22.1)
                    192.168.101.32
         ┌──────────────────────────────────┐
         │   nginx 反向代理                  │
         │   ├── /api/*  → lxc131:5106      │
         │   ├── /hubs/* → lxc131:5106 (WS) │
         │   └── /*      → Vue 静态文件      │
         └──────────────┬───────────────────┘
                        │ proxy_pass
         ┌──────────────▼───────────────────┐
         │   lxc131 (dotnet 8.0.27)          │
         │   192.168.101.31                  │
         │   └─ Kestrel http://0.0.0.0:5106  │
         └──────────────┬───────────────────┘
                        │ connect
         ┌──────────────▼───────────────────┐
         │   lxc137 (PostgreSQL 18.3)        │
         │   192.168.101.37                  │
         │   └─ database: eams2026           │
         └───────────────────────────────────┘

    HWATT SQL Server (外部/远程, 改连接串即可)
```

**组件说明：**

| 组件 | 技术栈 | 说明 |
|------|--------|------|
| 反向代理 | NGINX 1.22.1 | 统一入口，代理 API/WebSocket，托管前端静态文件 |
| 后端服务 | .NET 8.0.27 (Kestrel) | REST API + SignalR，监听 `http://0.0.0.0:5106` |
| 数据库 | PostgreSQL 18.3 | 主数据库 `eams2026` |
| 外部数据源 | SQL Server (远程) | HWATT 数据导入来源 |

**数据流向：**

1. 用户请求 → `http://192.168.101.32:8266` (Nginx)
2. `/api/*` 和 `/hubs/*` → 反向代理 → lxc131:5106 (.NET API)
3. 静态资源 (`/*`) → Nginx 直接返回 Vue 打包文件
4. .NET API → lxc137:5432 (PostgreSQL)

### 1.3 网络拓扑

```
公网/内网用户
     │
     ▼
lxc132 :8266  (Nginx 反向代理，对外暴露)
     │
     ├── /api/*   ──→ lxc131:5106 (内网，防火墙限制)

     └── /*       ──→ 本地静态文件
```

## 2. 部署步骤

### 2.1 PostgreSQL 部署（lxc137 — 192.168.101.37）

#### 2.1.1 安装 PostgreSQL 18.3

```bash
# 添加 PostgreSQL 官方源（Debian 12）
apt update && apt install -y curl gnupg2 lsb-release
sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'
curl -fsSL https://www.postgresql.org/media/keys/ACCC4CF8.asc | gpg --dearmor -o /etc/apt/trusted.gpg.d/postgresql.gpg
apt update

# 查看可用版本，确认 18 存在
apt-cache search postgresql-18

# 安装 PostgreSQL 18
apt install -y postgresql-18
```

#### 2.1.2 配置远程访问

```bash
# 编辑 postgresql.conf，监听所有网卡
nano /etc/postgresql/18/main/postgresql.conf
```

```ini
listen_addresses = '*'          # 默认是 localhost
port = 5432
```

```bash
# 编辑 pg_hba.conf，允许 lxc131 连接
nano /etc/postgresql/18/main/pg_hba.conf
```

```conf
# 在文件末尾添加
host    eams2026    eams_user    192.168.101.31/32    md5
```

#### 2.1.3 创建数据库和用户

```bash
sudo -u postgres psql
```

```sql
CREATE USER eams_user WITH PASSWORD 'eams2026';
CREATE DATABASE eams2026 OWNER eams_user;
\c eams2026
```

#### 2.1.4 导入数据库

**在 Windows 开发机上导出：**

```powershell
& "D:\Scoop\apps\postgresql\current\bin\pg_dump.exe" `
  -h localhost `
  -U postgres `
  -d eams2026 `
  -F c `
  -f d:\Codes\Projects\eams2026\publish\eams2026_database.dump
```

```powershell
scp d:\Codes\Projects\eams2026\publish\eams2026_database.dump root@192.168.101.37:/tmp/
```

**在 lxc137 上恢复：**

```bash
pg_restore -U postgres -d eams2026 -v /tmp/eams2026_database.dump
```

> 如果不需要保留数据，也可以直接运行初始化 SQL 脚本。

#### 2.1.5 授予数据库权限

```bash
sudo -u postgres psql -d eams2026 -c "GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO eams_user;"
sudo -u postgres psql -d eams2026 -c "ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON TABLES TO eams_user;"
sudo -u postgres psql -d eams2026 -c "GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO eams_user;"
```

#### 2.1.6 重启服务

```bash
systemctl restart postgresql
systemctl enable postgresql

# 验证监听状态
ss -tlnp | grep 5432
```

```ini
WantedBy=multi-user.target
```

**在 lxc132 上：**


#### 2.3.3 配置 Nginx

```bash
nano /etc/nginx/sites-available/eams2026
```

```nginx
server {
    listen 80;
    server_name _;

    root /opt/eams2026-web;
    index index.html;

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml text/javascript image/svg+xml;
    gzip_min_length 1000;

    # API 反向代理 → lxc131
    location /api/ {
        proxy_pass http://192.168.101.31:5106;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # SignalR WebSocket 代理 → lxc131
    location /hubs/ {
        proxy_pass http://192.168.101.31:5106;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400;
    }

    # SPA 路由
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

```bash
# 启用站点
ln -s /etc/nginx/sites-available/eams2026 /etc/nginx/sites-enabled/

# 测试配置
nginx -t

# 重启
systemctl restart nginx
systemctl enable nginx
```

## 6. 故障排查

### 6.1 API 返回 500 Internal Server Error

#### 症状

浏览器登录时提示"服务器内部错误，请稍后重试"，控制台显示：

```
POST http://192.168.101.32:8266/api/auth/login 500 (Internal Server Error)
```

#### 排查步骤

```bash
# 1. 查看后端日志
tail -50 /var/log/eams2026/eams2026-*.log

# 2. 确认数据库可连通
psql -h 192.168.101.37 -U eams_user -d eams2026 -c "SELECT 1;"

# 3. 测试网络连通性
nc -zv 192.168.101.37 5432
```

### 6.2 数据库权限不足

#### 症状

后端日志显示：

```
MessageText: 对表 users 权限不够
SqlState: 42501
Routine: aclcheck_error
```

```
对于序列 operation_logs_id_seq, 权限不够
```

#### 原因

- ✅ 数据库连接成功（用户名密码正确）
- ✅ 数据库已导入（表结构存在）
- ❌ `eams_user` 对表和序列缺少 INSERT/UPDATE/SELECT 权限

#### 解决方案

在 **lxc137**（PostgreSQL 服务器）上执行：

```bash
sudo -u postgres psql -d eams2026 -c "GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO eams_user;"
sudo -u postgres psql -d eams2026 -c "ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON TABLES TO eams_user;"
sudo -u postgres psql -d eams2026 -c "GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO eams_user;"
```

然后重启服务：

```bash
systemctl restart postgresql@18-main
```

> **注意：** 如果 `sudo -u postgres psql` 提示对等认证失败，说明 `pg_hba.conf` 中本地连接使用 `peer` 认证方式，这是正常行为。使用 `sudo -u postgres` 即可正确连接。

### 6.3 数据库连接被拒绝

#### 症状

PostgreSQL 日志显示：

```
用户 "postgres" Password 认证失败
用户 "postgres" 没有分配密码.
```

#### 原因

API 连接字符串中使用了 `Username=postgres`，但 `postgres` 用户没有设置密码。应使用专用用户 `eams_user`。

#### 解决方案

检查并确认连接字符串使用正确的数据库用户：

```json
"DefaultConnection": "Host=192.168.101.37;Port=5432;Database=eams2026;Username=eams_user;Password=eams2026"
```

### 6.4 Nginx 无法访问

#### 症状

浏览器访问 `http://192.168.101.32` 无响应或显示 502/503

#### 排查步骤

```bash
# 1. 检查 Nginx 状态
systemctl status nginx

# 2. 检查 Nginx 配置
nginx -t

# 3. 检查后端是否存活
curl -I http://192.168.101.31:5106/api/auth/login

# 4. 查看 Nginx 错误日志
tail -50 /var/log/nginx/error.log
```
