## 2. 部署步骤

### 2.1 数据库部署

#### 2.1.1 安装 PostgreSQL 18.3

**Windows：**
```powershell
# 使用 Scoop 安装
scoop bucket add versions
scoop install postgresql18

# 启动服务
pg_ctl start

# 设置开机自启
pg_ctl register -N PostgreSQL -D "$env:PGBASE\data"
```

**Ubuntu：**
```bash
# 添加 PostgreSQL 源
sudo sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'
curl -fsSL https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo gpg --dearmor -o /etc/apt/trusted.gpg.d/postgresql.gpg
sudo apt update
sudo apt install postgresql-18

# 启动服务
sudo systemctl enable postgresql
sudo systemctl start postgresql
```

#### 2.1.2 创建数据库

```sql
-- 连接 PostgreSQL
psql -U postgres

-- 创建数据库
CREATE DATABASE eams2026 WITH ENCODING 'UTF8' LC_COLLATE 'zh_CN.UTF-8' LC_CTYPE 'zh_CN.UTF-8';
-- 如果系统不支持 zh_CN.UTF-8，可使用:
CREATE DATABASE eams2026 WITH ENCODING 'UTF8';

-- 创建用户并授权
CREATE USER eams_admin WITH PASSWORD 'your_strong_password';
GRANT ALL PRIVILEGES ON DATABASE eams2026 TO eams_admin;
```

#### 2.1.3 执行数据库迁移

```powershell
# 方式一：使用 psql 逐一执行迁移脚本
$scripts = Get-ChildItem "src/EAMS2026.Infrastructure/Migrations/*.sql" | Sort-Object Name
foreach ($script in $scripts) {
    Write-Host "执行迁移: $($script.Name)"
    psql -h localhost -U postgres -d eams2026 -f $script.FullName
}

# 方式二：使用 pg_restore 导入备份
pg_restore -h localhost -U postgres -d eams2026 publish/eams2026_database.dump
```

### 2.2 后端部署 (Windows)

#### 2.2.1 发布

```powershell
# 设置 dotnet 路径
$dotnet = "D:\Scoop\apps\dotnet8-sdk\current\dotnet.exe"

# 发布 API
& $dotnet publish src/EAMS2026.Api -c Release -o publish/eams2026-api
```

#### 2.2.2 配置

创建/修改 `publish/eams2026-api/appsettings.json`：

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=localhost;Port=5432;Database=eams2026;Username=eams_admin;Password=your_strong_password"
  },
  "Jwt": {
    "SecretKey": "你的密钥，至少32位字符，请更换为随机字符串",
    "ExpireMinutes": 1440
  },
  "Serilog": {
    "WriteTo": [
      { "Name": "Console" },
      {
        "Name": "File",
        "Args": {
          "path": "logs/eams2026-.log",
          "rollingInterval": "Day",
          "retainedFileCountLimit": 30
        }
      }
    ]
  },
  "Cors": {
    "AllowedOrigins": ["http://localhost:8266", "http://192.168.101.32:8266"]

```

#### 2.2.3 创建 Windows 服务

```powershell
# 启动服务
sc.exe start EAMS2026

# 查看状态
sc.exe query EAMS2026
```

#### 2.2.4 手动启动（调试）

```powershell
# 直接运行
.\publish\eams2026-api\EAMS2026.Api.exe

# 或使用 dotnet run

```

### 2.3 后端部署 (Ubuntu)

#### 2.3.1 安装 .NET Runtime

```bash
# 注册 Microsoft 包源
wget https://packages.microsoft.com/config/ubuntu/22.04/packages-microsoft-prod.deb -O packages-microsoft-prod.deb
sudo dpkg -i packages-microsoft-prod.deb
sudo apt update

# 安装 .NET 8 Runtime
sudo apt install -y aspnetcore-runtime-8.0
```

#### 2.3.2 发布和部署

```bash
# 发布 (在开发机上)
dotnet publish src/EAMS2026.Api -c Release -o publish/eams2026-api -r linux-x64 --self-contained false

# 复制到服务器
scp -r publish/eams2026-api/* user@server:/var/www/eams/api/

# 创建 systemd 服务
sudo tee /etc/systemd/system/eams-api.service > /dev/null <<EOF
[Unit]
Description=EAMS2026 API
After=network.target postgresql.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/eams/api
ExecStart=/usr/bin/dotnet /var/www/eams/api/EAMS2026.Api.dll
Restart=always
RestartSec=10
Environment=ASPNETCORE_ENVIRONMENT=Production

[Install]
WantedBy=multi-user.target
EOF

# 启动服务
sudo systemctl daemon-reload
sudo systemctl enable eams-api
sudo systemctl start eams-api
```

### 2.5 Nginx 配置

#### 2.5.1 安装 Nginx

**Windows：**
```powershell
scoop install nginx
```

**Ubuntu：**
```bash
sudo apt install nginx
```

#### 2.5.2 配置反向代理

```nginx
# /etc/nginx/sites-available/eams2026 或 C:\EAMS2026\nginx\conf.d\eams2026.conf

upstream eams_api {
    server 127.0.0.1:5106;
    keepalive 32;
}

server {
    listen 8266;
    server_name _;

    # 前端静态文件
    root C:/EAMS2026/www;  # Windows
    # root /var/www/eams/web;  # Ubuntu
    index index.html;

    # 字符编码
    charset utf-8;

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;

    # API 反向代理
    location /api/ {
        proxy_pass http://eams_api;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection keep-alive;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 90s;
    }

    # SignalR WebSocket 代理
    location /hubs/ {
        proxy_pass http://eams_api;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 86400s;
    }

    # Swagger（仅开发环境开启）
    # location /swagger/ {
    #     proxy_pass http://eams_api;
    # }

    # 前端路由（SPA 所有路径返回 index.html）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location /assets/ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

#### 2.5.3 启动 Nginx

```powershell
# Windows
nginx -c C:\EAMS2026\nginx\conf.d\eams2026.conf
nginx -s reload

# Ubuntu
sudo ln -s /etc/nginx/sites-available/eams2026 /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

## 3. 配置说明

### 3.1 appsettings.json 核心配置

```json
{
  // 数据库连接
  "ConnectionStrings": {
    "DefaultConnection": "Host=localhost;Port=5432;Database=eams2026;Username=xxx;Password=xxx",
    "HwattConnection": "Server=db;Database=HWATT;User ID=sa;Password=xxx;TrustServerCertificate=True"
  },

  // JWT 认证
  "Jwt": {
    "SecretKey": "至少32位随机字符",
    "Issuer": "EAMS2026",
    "Audience": "EAMS2026",
    "ExpireMinutes": 1440   // Token 有效期（分钟）
  },

  // 钉钉考勤同步
  "DingTalk": {
    "AppKey": "",
    "AppSecret": "",
    "AttendanceGroupName": "考勤组",
    "AttendanceBatchDays": 7
  },

  // HWATT 考勤同步
  "HwattImport": {
    "BrchId": 3,
    "DefaultSyncStartDate": "2024-01-01",
    "BatchDays": 7
  }
}
```

### 3.2 环境变量覆盖

敏感配置可通过环境变量覆盖（优先级高于配置文件）：

```bash
# Linux
export ConnectionStrings__DefaultConnection="Host=prod-db;..."
export Jwt__SecretKey="production-secret-key"

# Windows PowerShell
$env:ConnectionStrings__DefaultConnection = "Host=prod-db;..."
```

## 6. 故障排查

### 6.1 API 启动失败

| 原因 | 排查 | 解决 |
|------|------|------|
| 端口被占用 | `netstat -ano | findstr :5106` | 更换端口或终止占用进程 |
| 数据库连接失败 | 检查 `appsettings.json` 连接字符串 | 确认数据库服务运行中 |
| JWT 密钥未配置 | 检查日志中 "SecretKey 未设置" 错误 | 在配置文件中设置 SecretKey |

### 6.2 前端访问白屏

| 原因 | 排查 | 解决 |
|------|------|------|
| Nginx 配置错误 | `nginx -t` | 修正配置文件语法 |
| 静态文件路径错误 | 检查 Nginx root 路径 | 确认 dist 文件已复制到正确目录 |
| 浏览器控制台错误 | F12 查看 Network 和 Console | 根据错误信息排查 |

### 6.3 报表执行慢

| 原因 | 解决 |
|------|------|
| SQL 查询效率低 | 优化 SQL，添加必要索引 |
| 数据量过大 | 缩小查询范围，使用分页 |
| 数据库连接问题 | 检查数据库服务器负载 |
