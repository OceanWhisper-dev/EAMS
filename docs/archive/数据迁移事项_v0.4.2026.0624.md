## 1. 数据库架构说明

### 1.1 数据库概览

| 数据库 | 类型 | 用途 | 连接配置 |
|--------|------|------|---------|
| `eams2026` | PostgreSQL 18 | 主数据库 — 系统管理、考勤、报表、ERP 设置等 | `Host=localhost;Port=5432;Database=eams2026;Username=postgres;Password=postgres` |
| `HWATT` | SQL Server 2005 | 遗留打卡数据源（只读） | `Server=db;Database=HWATT;User ID=sa;Password=dfl_DB;TrustServerCertificate=True` |
| `UFDATA_008_{year}` | SQL Server (U8 ERP) | U8 ERP 业务数据库，按年份分库 | 通过 `erp_settings_datasource` 表动态读取连接字符串 |

### 1.2 连接工厂结构

系统使用三个连接工厂，分别对应上述数据库：

| 连接工厂 | 目标数据库 | 实现技术 | 特殊说明 |
|---------|-----------|---------|---------|
| `DbConnectionFactory` | eams2026 (PostgreSQL) | Npgsql + Dapper | 注册了 DateOnly/TimeOnly 类型处理器；启用 `Npgsql.EnableLegacyTimestampBehavior` |
| `HwattConnectionFactory` | HWATT (SQL Server) | Microsoft.Data.SqlClient | 自动追加 `Encrypt=False;TrustServerCertificate=True` 以兼容 SQL Server 2005 |
| `ErpConnectionFactory` | U8 ERP (SQL Server) | System.Data.SqlClient (Windows) / ODBC + FreeTDS (Linux) | 连接字符串从 `erp_settings_datasource` 表实时读取；支持按年份切换数据库 |

### 3.4 迁移注意事项

#### ⚠️ 执行顺序

迁移脚本必须严格按编号顺序执行。`036` 必须在 `035` 之后执行，因为 `036` 依赖于前面脚本创建的表结构。

#### ⚠️ 模块化重命名的影响

Migration 036/039 对表名进行了大规模重命名，**代码中所有 SQL 引用已在同一版本中同步更新为新表名**。如果在 v0.4.2026.0624 之前有自定义脚本或外部工具直接访问数据库，需更新表名引用：

| 旧表名 | 新表名 |
|--------|--------|
| `departments` | `sys_departments` |
| `users` | `sys_users` |
| `employees` | `sys_employees` |
| `roles` | `sys_roles` |
| `operation_logs` | `sys_operation_logs` |
| `hwatt_card_records` | `attendance_hwatt_card_records` |
| `dingtalk_card_records` | `attendance_dingtalk_card_records` |
| `rpt_report` | `erp_rpt_report` |
| `rpt_datasource` | `erp_settings_datasource` |
| `rpt_salesperson_map` | `erp_settings_salesperson_map` |

#### ⚠️ 幂等性

迁移脚本已包含 `IF NOT EXISTS` / `DROP TABLE IF EXISTS` / 条件检查等幂等处理，可重复执行同一脚本而不会报错。

#### ⚠️ 数据备份

在执行任何迁移前，请先备份数据库：

```sql
-- 全量备份（自定义格式）
pg_dump -h localhost -U postgres -d eams2026 -F c -f eams2026_before_migration_20260624.dump

-- SQL 格式备份
pg_dump -h localhost -U postgres -d eams2026 > eams2026_before_migration_20260624.sql
```

### 4.2 报表迁移

#### 迁移工具

`scripts/migrate-reports.ps1` 是报表专用迁移脚本，使用 PowerShell + ADO.NET 完成数据迁移。

辅助工具 `scripts/ReportMigrator/` 是一个独立的 .NET 8 控制台程序，提供数据源连接测试、迁移完整性验证等功能。

#### 数据映射关系

| EAMS_4.6 表 | EAMS2026 表 | 关键映射说明 |
|-------------|-------------|------------|
| Reports | `erp_rpt_report` | reportID → id, Name → name, Title → title, QueryBase → query_text |
| report_Class | `erp_rpt_category` | clsID → id, clsName → name, parentID → parent_id |
| report_Fields | `erp_rpt_field` | fieldID → id, fieldName → field_name, fieldTitle → field_title |
| report_Filter | `erp_rpt_filter` | filterID → id, fieldName → field_name, defaultValue → default_value |
| report_Order | `erp_rpt_sort` | orderID → id, fieldName → field_name |
| report_Permissions | `erp_rpt_permission` | AutoID → id, RoleID → role_id |
| report_PivotView | 部分 → `erp_rpt_chart` | 迁移配置到 chart_config JSON 字段 |

#### 迁移前准备

```powershell
# 1. 确保 SQL Server 源数据库可访问
# 2. 确保 PostgreSQL 目标数据库已执行全部迁移脚本（001~042）
# 3. 编辑 migrate-reports.ps1 中的连接字符串
#    源：SQL Server 的 AppData 数据库
#    目标：PostgreSQL 的 eams2026 数据库

$srcConnStr = "Server=db;Database=AppData;User ID=sa;Password=密码;TrustServerCertificate=True"
$dstConnStr = "Host=localhost;Port=5432;Database=eams2026;Username=postgres;Password=postgres"
```

#### 执行迁移

```powershell
cd scripts
.\migrate-reports.ps1
```

### 4.3 考勤数据迁移

#### 遗留打卡数据导入

考勤数据迁移通过两个渠道完成：

**渠道一：迁移脚本**
- `012_ImportLegacyData.sql` — 导入基础部门/员工数据
- `013_ImportUsersAndEmployees.sql` — 导入用户和员工关联数据
- `013_ImportLegacyEventDeclareds.sql` — 导入遗留事件声明数据

**渠道二：系统数据导入页面**
- 通过系统"数据导入"页面，选择"遗留系统"
- 支持按日期范围分批导入
- 实时查看导入进度（SignalR WebSocket 推送，Hub 路径 `/hubs/import`）
- 支持断点续传

#### 员工映射

HWATT 考勤员工号 ↔ 系统员工号的映射：

- 支持手动映射和自动映射（按工号/姓名匹配）
- 未匹配的员工可手动关联
- 映射数据存储在 `attendance_hwatt_employees` 表

#### 导入注意事项

```sql
-- 检查未映射的考勤员工
SELECT ae.* FROM attendance_hwatt_employees ae
LEFT JOIN sys_employees e ON e.employee_no = ae.employee_no
WHERE e.id IS NULL;

-- 检查重复的打卡记录
SELECT employee_no, card_time, COUNT(*)
FROM attendance_hwatt_card_records
GROUP BY employee_no, card_time
HAVING COUNT(*) > 1;
```

## 7. 验证检查清单

### 7.1 数据库完整性

```sql
-- 1. 检查所有模块表是否存在
SELECT table_name FROM information_schema.tables
WHERE table_schema = 'public'
ORDER BY table_name;

-- 2. 检查表行数
SELECT schemaname, tablename, n_live_tup
FROM pg_stat_user_tables
ORDER BY tablename;

-- 3. 检查序列是否同步
SELECT sequence_name, last_value
FROM information_schema.sequences
WHERE sequence_schema = 'public'
ORDER BY sequence_name;

-- 4. 检查外键完整性（孤儿记录检查）
-- 示例：检查报表是否有不存在的分类
SELECT COUNT(*) AS orphan_reports
FROM erp_rpt_report r
LEFT JOIN erp_rpt_category c ON c.id = r.category_id
WHERE r.category_id IS NOT NULL AND c.id IS NULL;
```

### C. 数据一致性检查 SQL 速查

```sql
-- 1. 所有表行数概览
SELECT schemaname, tablename, n_live_tup
FROM pg_stat_user_tables
ORDER BY n_live_tup DESC;

-- 2. 外键完整性
-- 报表引用不存在的分类
SELECT r.id, r.name FROM erp_rpt_report r
LEFT JOIN erp_rpt_category c ON c.id = r.category_id
WHERE r.category_id IS NOT NULL AND c.id IS NULL;

-- 3. 软删除数据量
SELECT 'sys_departments' AS tbl, COUNT(*) FROM sys_departments WHERE is_deleted = true
UNION ALL
SELECT 'sys_employees', COUNT(*) FROM sys_employees WHERE is_deleted = true
UNION ALL
SELECT 'sys_users', COUNT(*) FROM sys_users WHERE is_deleted = true;

-- 4. 序列修复检查
SELECT sequence_name, last_value
FROM information_schema.sequences
WHERE sequence_schema = 'public'
ORDER BY sequence_name;

-- 5. 模块表数量统计
SELECT
    CASE
        WHEN table_name LIKE 'sys\_%' THEN 'sys_* (系统管理)'
        WHEN table_name LIKE 'attendance\_%' THEN 'attendance_* (考勤)'
        WHEN table_name LIKE 'erp\_rpt\_%' THEN 'erp_rpt_* (报表)'
        WHEN table_name LIKE 'erp\_vouch\_modify\_%' THEN 'erp_vouch_modify_* (单据修改)'
        WHEN table_name LIKE 'erp\_settings\_%' THEN 'erp_settings_* (ERP设置)'
        ELSE '其他'
    END AS module,
    COUNT(*) AS table_count
FROM information_schema.tables
WHERE table_schema = 'public'
GROUP BY module
ORDER BY module;
```

### D. SQL 语法转换参考 (T-SQL → PostgreSQL)

| T-SQL (SQL Server) | PostgreSQL | 说明 |
|-------|-----------|------|
| `GETDATE()` | `NOW()` | 当前时间 |
| `ISNULL(val, default)` | `COALESCE(val, default)` | 空值处理 |
| `CHARINDEX(sub, str)` | `POSITION(sub IN str)` | 字符串查找 |
| `SUBSTRING(str, start, len)` | `SUBSTRING(str FROM start FOR len)` | 子串 |
| `LEN(str)` | `LENGTH(str)` | 字符串长度 |
| `DATEDIFF(day, d1, d2)` | `EXTRACT(DAY FROM d2 - d1)` | 日期差 |
| `TOP n` | `LIMIT n` | 限制行数 |
| `CAST(x AS type)` | `x::type` | 类型转换 |
| `CONVERT(type, val)` | `val::type` | 类型转换 |
| 方括号 `[col]` | 双引号 `"col"` | 标识符引用 |
