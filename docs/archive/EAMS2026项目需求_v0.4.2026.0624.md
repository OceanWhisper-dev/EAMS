## 2. 项目背景与目标

### 2.1 项目背景

EAMS2026（Enterprise Asset & Management System 2026）企业管理系统，旨在将企业中多个已有的 C# 程序（BS + CS 架构）功能整合重构为一个统一的企业管理系统。采用**先跑通、再优化**的策略，优先保证核心业务功能的可用性。

### 2.2 产品定位

- 面向中大型企业的内部管理系统
- 纯局域网部署，数据不出企业内网
- 模块化设计，可按需扩展
- 支持从现有 SQL Server 2005（HWATT）考勤系统和 U8 ERP 系统同步数据

### 2.3 项目目标

1. 整合多个遗留系统的功能于统一平台
2. 提供基于 RBAC 的细粒度权限控制
3. 支持多数据源考勤数据采集（HWATT 打卡机 + 钉钉 API）
4. 提供 U8 ERP 辅助功能（报表查询 + 单据修改）
5. 支持局域网多用户并发访问

### 2.4 用户角色

| 用户角色 | 说明 | 典型操作 |
|----------|------|----------|
| 超级管理员 | 拥有系统所有权限 | 系统配置、权限分配、数据管理 |
| 系统管理员 | 管理基础数据和用户 | 部门/员工/用户/角色/权限管理 |
| 考勤管理员 | 管理考勤模块 | 同步打卡数据、维护排班方案、生成考勤报表 |
| 报表管理员 | 设计和管理报表 | 创建/编辑 SQL 报表、配置数据源、管理权限 |
| ERP 操作员 | 使用 ERP 辅助功能 | 修改单据客户、批量调整发货日期 |
| 普通员工 | 使用系统基础功能 | 查看仪表盘、修改个人资料、收发消息 |

## 4. 各模块功能需求简述

### 4.1 系统管理模块

系统管理模块提供企业基础数据管理和系统运维能力，包括组织架构（部门/员工）、访问控制（用户/角色/权限）、数据字典、站内消息、仪表盘、数据权限、操作审计等 13 个子功能。所有核心实体支持软删除和完整的审计字段追踪。

### 4.2 考勤管理模块

考勤管理模块支持多数据源（HWATT SQL Server 打卡机和钉钉 API）的考勤数据采集、员工映射、排班管理（班次方案/计划时间/员工关联）、考勤计算（迟到/早退/缺卡/加班自动识别）、节假日管理、事件声明（加班/请假/出差/调休）、费用计算规则配置，以及月度考勤报表输出。

批量数据导入通过 SignalR WebSocket 实时推送处理进度，前端展示进度条。

### 4.3 ERP 辅助模块

**报表管理**：自定义 SQL 报表引擎，支持设计器在线编辑 SQL、参数化查询、服务端分页/排序、多数据源切换（PostgreSQL 主库 / SQL Server ERP）、图表展示（ECharts）、Excel 导出、透视表分析（PivotTable.js）、报表级和字段级权限控制。

**单据修改**：对 U8 ERP 的销售订单和发货单进行客户信息修改和发货日期调整（单笔和批量），自动记录修改日志，支持客户参照验证。

**ERP 设置**：数据源配置（报表和单据修改共用）和业务员映射管理。

## 5. 技术约束

### 5.1 SQL Server 2005 兼容

- HWATT 考勤数据源为 **SQL Server 2005**
- 连接时必须追加 `Encrypt=False;TrustServerCertificate=True;` 以禁用 TLS 加密协商（SQL Server 2005 不支持 TLS 1.0+）
- 使用 `Microsoft.Data.SqlClient`（Windows）和 `System.Data.Odbc` + FreeTDS（Linux）双平台兼容驱动
- ERP 数据库 U8 使用 `UFDATA_{账套号}_{年份}` 命名规则，支持按年份切换数据库

### 5.2 局域网部署

- 系统设计为纯局域网使用，API 和前端开发服务器均监听 `0.0.0.0` 允许局域网内访问
- 不支持公网 HTTPS 部署场景（当前版本）
- CORS 开发模式允许所有来源（`SetIsOriginAllowed(_ => true)`）
- Token 存储在 localStorage（内网低风险场景，生产环境建议改用 httpOnly Cookie）

### 5.3 其他约束

- 密码使用 BCrypt 哈希存储，JWT SecretKey ≥ 32 字符
- 所有数据库操作使用 Dapper 参数化 SQL 防注入
- 前端使用 Element Plus 中文语言包
- Vite 开发服务器代理 API 请求到后端 Kestrel（`http://localhost:5106`）
