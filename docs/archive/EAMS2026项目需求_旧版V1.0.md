## 第一部分：项目概述

### 1.1 项目名称
**EAMS2026**（Enterprise Asset & Approval Management System 2026）

### 1.2 项目目标
将现有的多个C#程序（BS+CS）功能整合重构为一个**局域网部署的BS架构**企业管理系统。采用**先跑通、再优化**的策略，后期可根据需要将高并发模块（如考勤同步）用Rust改写，但需提前做好接口抽象。

### 1.3 部署环境
- **操作系统**：Debian 12
- **Web服务器**：Nginx（反向代理 + 静态资源）
- **应用服务器**：.NET 8（Kestrel）
- **数据库**：PostgreSQL 18（主库）
- **外部数据源**：
  - SQL Server 2005（用友U8 V8.71 ERP）——**读写**
  - SQL Server 2012+（考勤机）——**只读**
  - 钉钉考勤API（新增）

### 1.4 技术栈
| 层级 | 技术 |
|------|------|
| 前端 | Vue 3 + Vite + TypeScript + Pinia + Vue Router + Element Plus |
| 后端 | .NET 8 Web API + Dapper（不使用EF Core） |
| 数据库 | PostgreSQL 18（主库），SQL Server（外部） |
| 认证 | JWT（Bearer Token） |
| 打印 | PuppeteerSharp（HTML → PDF） |
| 定时任务 | BackgroundService / Quart.NET |
| 依赖注入 | .NET 8 内置DI |

### 3.3 数据库连接配置（Debian 12）

#### SQL Server 2005（U8）—— 使用 FreeTDS
```bash
# 安装
sudo apt-get install freetds-bin freetds-dev tdsodbc

# /etc/freetds/freetds.conf
[U8ERP]
    host = <U8服务器IP>
    port = 1433
    tds version = 7.2
    client charset = UTF-8

# 测试
tsql -S U8ERP -U <user> -P <pwd>
```

#### C# 连接字符串
```csharp
// SQL Server 2005
"Server=<IP>;Database=U8Data;User Id=<user>;Password=<pwd>;Encrypt=false;"

// SQL Server 2012（考勤机）
"Server=<IP>;Database=Attendance;User Id=<user>;Password=<pwd>;Encrypt=false;"

// PostgreSQL 18
"Host=localhost;Database=eams2026;Username=eams_user;Password=<pwd>;Port=5432;"
```

### 3.4 SQL Server 2005 语法限制（关键！）
| 禁止使用的特性 | 替代方案 |
|--------------|----------|
| `OFFSET / FETCH` | 使用 `ROW_NUMBER() OVER(...)` 分页 |
| `OUTPUT` 子句 | 先查询再更新，或使用 `SCOPE_IDENTITY()` |
| `SNAPSHOT` 隔离级别 | 使用 `READ COMMITTED` |
| `TRY...CATCH` 中的某些语法 | 使用标准 `RAISERROR` |
| `CONCAT` 函数 | 使用 `+` 运算符 |

## 第六部分：Trae CN 操作流程与注意事项

### 6.1 基本操作流程
1. **打开Trae CN**，创建/打开项目文件夹。
2. **输入提示词**（可从文档第五部分复制）。
3. **按 `Ctrl+Enter`** 提交。
4. **检查输出**：
   - 符合预期 → 复制到对应文件。
   - 不符合 → 补充约束条件（如“不要使用OFFSET”），重新生成。
5. **集成测试** → 继续下一个提示词。

### 6.2 代码修改流程
1. 选中需要修改的代码块。
2. 点击悬浮菜单中的“添加到对话”。
3. 描述修改需求（例如：“将这个SQL改为兼容SQL Server 2005”）。
4. 复制Trae CN输出的修改后代码，替换原代码。

### 6.3 重要注意事项

| 情况 | 应对方法 |
|------|----------|
| Trae生成了`OFFSET/FETCH` | 在每个SQL提示词中强调“**必须兼容SQL Server 2005，禁止OFFSET/FETCH**” |
| Trae忘记之前的约束 | 每个提示词都**重复关键约束**，不要假设它记住上下文 |
| 输出不完整 | 使用分步策略：“第一步做X，第二步做Y，现在先做第一步” |
| 引入未声明的库（如EF Core） | 明确写：“使用Dapper，不要引入Entity Framework Core” |
| 打印样式不匹配 | 提供Word样式的详细描述或截图作为参考 |
| 对话过长导致响应中断 | 分模块新建对话，每个功能模块单独开始 |
