## 1. 系统登录

### 操作步骤

1. 浏览器打开 `http://<服务器IP>:5173`（本机为 `http://localhost:5173`）
2. 输入用户名和密码
3. 点击「登录」按钮或按回车键

### 首次登录

- 超级管理员默认账号: 用户名和密码由初始化脚本设置
- 首次登录或密码重置后，系统会要求强制修改密码
- 密码长度不少于6位

### 参数说明

| 参数 | 必填 | 说明 |
|------|------|------|
| 用户名 | 是 | 系统分配的用户名 |
| 密码 | 是 | 至少6位字符 |

### 异常情况

| 场景 | 系统表现 |
|------|----------|
| 用户名不存在 | 提示"登录失败" |
| 密码错误 | 提示"登录失败" |
| 账户被禁用 | 提示"登录失败" |
| Token过期 | 自动跳回登录页 |

## 2. 仪表盘

### 功能说明

仪表盘是登录后的首页，展示系统的统计概览。

### 界面组成

- **统计卡片**: 显示部门数、员工数、用户数等关键指标
- **最近操作日志**: 以表格列表形式显示最近的系统操作记录，包含操作人、模块、操作类型、描述、时间

### 操作步骤

1. 登录后自动进入仪表盘
2. 通过左侧菜单点击「仪表盘」也可进入
3. 统计卡片数据自动刷新

## 3. 部门管理

### 功能说明

管理企业的组织架构树。支持无限层级的部门结构。

### 界面布局

- **左侧**: 部门树形列表，可展开/折叠
- **右侧**: 选中部门的详情和子部门列表

### 操作流程

#### 新增部门
1. 点击「新增」按钮
2. 填写以下字段：

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 部门名称 | 是 | 部门的显示名称 | `技术部` |
| 部门编码 | 是 | 唯一标识，不可重复 | `tech` |
| 上级部门 | 否 | 从下拉树中选择上级部门 | `总公司` |
| 描述 | 否 | 补充说明 | `负责产品研发` |
| 排序号 | 否 | 数字越小越靠前，默认0 | `1` |

3. 点击「保存」

#### 编辑部门
1. 在左侧树中点击要编辑的部门
2. 点击「编辑」按钮
3. 修改字段后点击「保存」

#### 删除部门
1. 在左侧树中点击要删除的部门
2. 点击「删除」按钮
3. 确认删除

> ⚠️ 注意：删除是软删除（标记删除），不会真正删除数据库记录。但删除前请确认该部门下无员工关联。

### 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| name | string | 部门名称，最长200字符 |
| code | string | 部门编码，最长50字符，必须唯一 |
| parent_id | number/null | 上级部门ID，null表示顶级部门 |
| description | string | 描述文本 |
| sort_order | number | 排序号，默认0 |

## 4. 员工管理

### 功能说明

管理企业员工信息。支持列表查询、Excel批量导入导出。

### 操作流程

#### 查看员工列表
1. 点击左侧菜单「员工管理」
2. 列表显示所有员工，包含：员工编号、姓名、性别、电话、邮箱、部门、职位、入职日期
3. 可通过部门筛选员工

#### 新增员工
1. 点击「新增」按钮
2. 填写字段：

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 员工编号 | 是 | 唯一编号，不可重复 | `EMP001` |
| 姓名 | 是 | 员工姓名 | `张三` |
| 性别 | 否 | 男/女 | `男` |
| 电话 | 否 | 联系电话 | `13800138000` |
| 邮箱 | 否 | 电子邮箱 | `zhangsan@example.com` |
| 所属部门 | 否 | 下拉选择部门 | `技术部` |
| 职位 | 否 | 岗位名称 | `软件工程师` |
| 入职日期 | 否 | 日期选择 | `2026-01-01` |
| 排序号 | 否 | 排序号，默认0 | `1` |

3. 点击「保存」

#### Excel导入
1. 点击「导入」按钮
2. 选择 `.xlsx` 格式的 Excel 文件
3. 系统自动解析并导入
4. 导入结果显示成功/失败数量

**Excel 模板要求**:
- 第一行为表头，必须包含: 员工编号、姓名、性别、电话、邮箱、部门名称、职位、入职日期
- 部门名称会自动匹配系统中已有的部门

#### Excel导出
1. 点击「导出」按钮
2. 系统下载 `.xlsx` 文件

#### 编辑/删除
- 在列表中点击「编辑」或「删除」按钮操作
- 删除为软删除

### 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| employee_no | string | 员工编号，最长50字符，必须唯一 |
| name | string | 姓名，最长100字符 |
| gender | string | 性别 |
| phone | string | 电话，最长20字符 |
| email | string | 邮箱，最长200字符 |
| department_id | number/null | 部门ID |
| position | string | 职位，最长200字符 |
| hire_date | date | 入职日期，格式YYYY-MM-DD |
| sort_order | number | 排序号 |

## 5. 用户管理

### 功能说明

管理系统登录账户。用户可关联员工信息，分配角色（角色决定权限）。

### 操作流程

#### 查看用户列表
1. 点击左侧菜单「用户管理」
2. 列表显示用户（支持分页和关键词搜索）
3. 每行显示：用户名、关联员工、角色、状态、最后登录时间

#### 新增用户
1. 点击「新增」按钮
2. 填写字段：

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 用户名 | 是 | 登录账号，不可重复 | `zhangsan` |
| 密码 | 是 | 登录密码，至少6位 | `123456` |
| 关联员工 | 否 | 下拉选择员工 | `张三` |
| 状态 | 否 | 启用/禁用 | 默认启用 |

3. 点击「保存」

#### 分配角色
1. 在用户列表中点击「分配角色」
2. 在弹出框中勾选角色（可多选）
3. 点击「保存」

> 📌 角色决定用户能看到哪些菜单、执行哪些操作。分配 `super_admin` 角色即拥有所有权限。

#### 重置密码
1. 在用户列表中点击「重置密码」
2. 确认后密码被重置为空
3. 该用户下次登录必须修改密码

#### 编辑/删除
- 超级管理员（id=1）不可删除
- 其他用户可编辑和软删除

### 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| username | string | 用户名，最长100字符，必须唯一 |
| password_hash | string | 密码明文，服务端自动BCrypt加密存储 |
| employee_id | number/null | 关联员工ID |
| status | boolean | 账户启用状态，true=启用 |
| force_change_password | boolean | 是否强制修改密码 |

## 7. 权限管理

### 功能说明

管理系统中的权限项。权限分为三种类型。

### 权限类型

| 类型 | 说明 | 使用场景 |
|------|------|----------|
| menu | 菜单权限 | 控制左侧导航菜单的显示/隐藏 |
| button | 按钮权限 | 控制按钮的显示/隐藏（通过 v-permission 指令） |
| api | 接口权限 | 控制后端API的访问 |

### 操作流程

#### 权限树结构
- 权限以树形结构组织，支持父子层级
- 父级权限通常是菜单，子级可以是按钮或API

#### 新增权限
1. 点击「新增」按钮
2. 填写字段：

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 权限名称 | 是 | 显示名称 | `用户管理` |
| 权限编码 | 是 | 唯一标识，与前端菜单码一致 | `user` |
| 权限类型 | 是 | menu/button/api | `menu` |
| 上级权限 | 否 | 树形父节点 | `系统管理` |
| 排序号 | 否 | 排序号 | `1` |

3. 点击「保存」

> ⚠️ 权限编码必须与以下规则一致：
> - 菜单级权限编码 = 前端路由中的权限码（如 `user`、`role`、`department`）
> - 按钮级权限编码 = 模块:操作格式（如 `user:create`、`user:delete`）
> - 后端 `[Authorize(Policy = "编码")]` 校验此编码

### 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| name | string | 权限名称，最长200字符 |
| code | string | 权限编码，最长50字符，必须唯一 |
| type | string | menu/button/api |
| parent_id | number/null | 父权限ID |
| sort_order | number | 排序号 |

## 8. 字典管理

### 功能说明

管理系统中的字典数据（下拉选项）。这是**两级结构**：先定义字典类型（类别），再定义该类型下的字典项（选项）。

### 界面布局

- **左侧**: 字典类型列表
- **右侧**: 选中类型下的字典项列表

### 操作流程

#### 第一步：新增字典类型
1. 在左侧点击「新增字典类型」
2. 填写：

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 名称 | 是 | 类型显示名称 | `员工状态` |
| 编码 | 是 | 唯一标识，代码通过此编码获取字典项 | `employee_status` |
| 描述 | 否 | 补充说明 | `用于员工状态筛选` |

#### 第二步：新增字典项
1. 在左侧选中刚创建的字典类型
2. 在右侧点击「新增项」
3. 填写：

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 标签 | 是 | 页面显示的文字 | `在职` |
| 值 | 是 | 程序存储的值 | `active` |
| 颜色 | 否 | Element Plus标签颜色 | `success`/`warning`/`danger`/`info` |
| 排序 | 否 | 数字越小越靠前 | `1` |
| 默认 | 否 | 是否默认选中 | 开关 |
| 状态 | 否 | 启用/禁用 | 默认启用 |

### 在代码中使用

```javascript
// 前端获取字典数据（用于下拉框等）
const res = await dictApi.getItemsByCode('employee_status')
// 返回: [{ label:'在职', value:'active' }, { label:'离职', value:'resigned' }, ...]
```

```html
<!-- 在下拉框中使用 -->
<el-select v-model="form.status">
  <el-option v-for="item in statusOptions" :key="item.value"
    :label="item.label" :value="item.value" />
</el-select>
```

### 完整示例：创建"性别"字典

| 步骤 | 操作 | 内容 |
|------|------|------|
| 1 | 新增类型 | 名称=性别, 编码=gender |
| 2 | 新增项1 | 标签=男, 值=male, 排序=1 |
| 3 | 新增项2 | 标签=女, 值=female, 排序=2 |

### 参数说明

**字典类型**
| 参数 | 类型 | 说明 |
|------|------|------|
| name | string | 类型名称 |
| code | string | 类型编码，唯一标识，代码中的查询键 |
| description | string | 描述 |

**字典项**
| 参数 | 类型 | 说明 |
|------|------|------|
| dict_type_id | number | 所属字典类型ID |
| label | string | 显示标签 |
| value | string | 存储值 |
| color | string/null | 显示颜色(success/warning/danger/info) |
| is_default | boolean | 是否默认项 |
| sort_order | number | 排序号 |
| status | boolean | 启用/禁用 |

## 9. 操作日志

### 功能说明

自动记录系统中所有成功的增删改操作。用于审计追溯。

### 界面说明

- 列表显示：操作人、模块、操作类型、描述、实体类型、IP地址、操作时间
- 支持筛选：按用户、模块、日期范围筛选
- 支持分页

### 操作

- **查看日志**: 点击菜单进入，系统自动加载
- **筛选**: 选择用户/模块/日期范围后自动过滤
- **查看个人日志**: 点击「我的日志」标签页
- **清除日志**: 管理员可清除指定日期之前的日志

### 自动记录的模块

| 模块 | 对应Controller | 操作类型 |
|------|---------------|----------|
| User | UserController | Create/Update/Delete |
| Role | RoleController | Create/Update/Delete |
| Department | DepartmentController | Create/Update/Delete |
| Employee | EmployeeController | Create/Update/Delete |
| Permission | PermissionController | Create/Update/Delete |
| Dict | DictController | Create/Update/Delete |
| Profile | AuthController | ChangePassword/Update |
| *(其他)* | OperationLogMiddleware | Query/Create/Update/Delete |

## 10. 仪表盘配置

### 功能说明

配置仪表盘中显示的组件（统计卡片、列表、图表等）。

### 操作流程

#### 管理组件
1. 点击菜单「仪表盘配置」
2. Tab页签「组件管理」显示所有组件

#### 新增组件
1. 点击「新增」按钮
2. 填写字段：

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 组件标识 | 是 | 唯一标识 | `user_stats_card` |
| 组件名称 | 是 | 显示名称 | `用户统计卡片` |
| 组件类型 | 是 | stat_card/table/chart/list | `stat_card` |
| 描述 | 否 | 组件说明 | `显示用户总数和状态分布` |
| 图标 | 否 | Element图标名 | `Avatar` |
| 数据源类型 | 是 | 数据来源 | 见下方说明 |
| 数据源配置 | 否 | JSON格式配置 | 见下方说明 |
| 刷新间隔 | 否 | 自动刷新秒数，0=不刷新 | `30` |
| 排序 | 否 | 数字越小越靠前 | `1` |

#### 数据源类型与配置

| 数据源类型 | 用途 | 配置格式 | 示例 |
|-----------|------|----------|------|
| `count` | 统计表记录数 | `{"table":"表名","condition":"WHERE条件"}` | `{"table":"users","condition":"is_deleted = FALSE"}` |
| `sql` | 执行自定义SQL | `{"sql":"SELECT语句","parameters":{}}` | `{"sql":"SELECT * FROM users WHERE status = TRUE"}` |
| `api` | 调用后端API | `{"url":"完整API地址"}` | `{"url":"http://localhost:5106/api/dashboard/stats/departments"}` |
| `recent_logs` | 最近操作日志 | `{"limit":10}` | `{"limit":5}` |
| `unread_messages` | 未读消息数 | `{}` | `{}` |

> ⚠️ 数据源配置必须是**合法的JSON格式**，用 `{}` 包裹。SQL类型不能直接写SQL语句，要写成 `{"sql":"你的SQL"}`。

#### 预览数据
1. 在新增/编辑组件弹窗中填写数据源类型和配置后
2. 点击「预览数据」按钮
3. 系统执行数据源查询并显示结果

#### 角色仪表盘配置
1. Tab页签「角色配置」
2. 选择角色
3. 从左侧组件池拖拽组件到仪表盘布局
4. 点击「保存」

## 附录B: 超级管理员特殊规则

- `roles.code = 'super_admin'` 的角色拥有所有权限
- 前端 `hasPermission()` 中 `isAdmin` 返回 true 时不检查具体权限
- 后端 `PermissionHandler` 中 `super_admin` 角色直接通过
- 用户 id=1（超级管理员）不可删除
