### 2.3 迁移注意事项

#### ⚠️ 执行顺序
迁移脚本必须严格按编号顺序执行。编号相同的脚本（如 `010_*`）可视为同一批次，但建议按字母顺序执行。

#### ⚠️ 幂等性
迁移脚本已包含 `IF NOT EXISTS` / `DROP TABLE IF EXISTS` 等幂等处理，可重复执行同一脚本而不会报错。

#### ⚠️ 数据备份
在执行任何迁移前，请先备份数据库：

```sql
-- 全量备份
pg_dump -h localhost -U postgres -d eams2026 > eams2026_before_migration_20260611.sql

-- 仅备份数据
pg_dump -h localhost -U postgres -d eams2026 --data-only > eams2026_data_20260611.sql
```

### 3.2 报表迁移

#### 迁移工具

`scripts/migrate-reports.ps1` 是报表专用迁移脚本，使用 PowerShell + ADO.NET 完成数据迁移。

#### 数据映射关系

| EAMS_4.6 表 | EAMS2026 表 | 关键映射说明 |
|-------------|-------------|------------|
| Reports | rpt_report | reportID → id, Name → name, Title → title, QueryBase → query_text |
| report_Class | rpt_category | clsID → id, clsName → name, parentID → parent_id |
| report_Fields | rpt_field | fieldID → id, fieldName → field_name, fieldTitle → field_title |
| report_Filter | rpt_filter | filterID → id, fieldName → field_name, defaultValue → default_value |
| report_Order | rpt_sort | orderID → id, fieldName → field_name |
| report_Permissions | rpt_permission | AutoID → id, RoleID → role_id |
| report_PivotView | 部分 → rpt_chart | 迁移配置到 chart_config JSON 字段 |

#### 迁移前准备

```powershell
# 1. 确保 SQL Server 源数据库可访问
# 2. 确保 PostgreSQL 目标数据库已执行 001~023 迁移脚本

$dstConnStr = "Host=localhost;Port=5432;Database=eams2026;Username=postgres;Password=postgres"
```

#### 执行迁移

```powershell
cd scripts
```


### 3.3 考勤数据迁移

#### 遗留考勤数据导入

考勤数据通过迁移脚本 012~013 和专门的导入功能完成。

**迁移流程：**

1. **导入部门和员工**（`012_ImportLegacyData.sql`、`013_ImportUsersAndEmployees.sql`）
   - 从 SQL Server 读取遗留部门/员工数据
   - 写入 PostgreSQL 目标表
   - 建立用户-员工关联关系

2. **打卡记录同步**
   - 通过系统"数据导入"页面，选择"遗留系统"
   - 支持按日期范围分批导入
   - 实时查看导入进度（SignalR 推送）

3. **员工映射**
   - HWATT 考勤员工号 ↔ 系统员工号
   - 支持手动映射和自动映射（按工号/姓名匹配）
   - 未匹配的员工可手动关联

#### 导入注意事项

```sql
-- 检查未映射的考勤员工
SELECT ae.* FROM attendance_employees ae
LEFT JOIN employees e ON e.employee_no = ae.employee_no
WHERE e.id IS NULL;

-- 检查重复的打卡记录
SELECT employee_no, card_time, COUNT(*)
FROM hwatt_card_records
GROUP BY employee_no, card_time
HAVING COUNT(*) > 1;
```

## 4. 报表数据迁移详解

### 4.1 字段类型映射

| EAMS_4.6 类型 | EAMS2026 类型 | 说明 |
|---------------|--------------|------|
| string / text | string | 文本类型 |
| int / float / double | number | 数值类型 |
| datetime / date | date | 日期类型 |
| bit / boolean | boolean | 布尔类型 |
| money / decimal | money | 货币类型（格式化为千位分隔） |

### 4.2 SQL 语法转换

EAMS_4.6 的 SQL 查询基于 SQL Server (T-SQL)，迁移到 EAMS2026 需要转换为 PostgreSQL 语法：

| T-SQL | PostgreSQL | 说明 |
|-------|-----------|------|
| `GETDATE()` | `NOW()` | 当前时间 |
| `ISNULL(val, default)` | `COALESCE(val, default)` | 空值处理 |
| `CHARINDEX(sub, str)` | `POSITION(sub IN str)` | 字符串查找 |
| `SUBSTRING(str, start, len)` | `SUBSTRING(str FROM start FOR len)` | 子串 |
| `LEN(str)` | `LENGTH(str)` | 字符串长度 |
| `DATEDIFF(day, d1, d2)` | `EXTRACT(DAY FROM d2 - d1)` | 日期差 |
| `TOP n` | `LIMIT n` | 限制行数 |
| `CAST(x AS type)` | `x::type` | 类型转换 |
| `CONVERT(type, val)` | `val::type` | 类型转换 |
| `NOCOUNT ON` | 不需要 | 控制消息 |
| 方括号 `[col]` | 双引号 `"col"` | 标识符引用 |

### 4.3 ReportMigrator 工具

`scripts/ReportMigrator/` 是一个独立的 .NET 控制台程序，用于更复杂的数据迁移：

```powershell
cd scripts/ReportMigrator
```

该工具提供了测试数据源连接、验证迁移完整性等功能。

## 6. 迁移注意事项

### 6.1 通用注意事项

1. **字符编码**：确保 PostgreSQL 数据库使用 UTF-8 编码
2. **序列重置**：迁移后检查序列值是否正确：

```sql
-- 检查并重置序列
SELECT setval('users_id_seq', (SELECT MAX(id) FROM users));
SELECT setval('departments_id_seq', (SELECT MAX(id) FROM departments));
SELECT setval('employees_id_seq', (SELECT MAX(id) FROM employees));
SELECT setval('rpt_report_id_seq', (SELECT MAX(id) FROM rpt_report));
```

3. **外键约束**：按依赖顺序导入数据（先导入父表，再导入子表）
4. **索引重建**：大量数据导入后重建索引：

```sql
REINDEX DATABASE eams2026;
ANALYZE;
```

### 6.2 报表迁移专项注意事项

1. **SQL 语法兼容**：EAMS_4.6 中可能使用了 SQL Server 特有的语法，迁移到 PostgreSQL 需手动调整
2. **参数语法**：EAMS_4.6 使用 `@param` 参数，EAMS2026 同样使用 `@param` 语法（Dapper 统一）
3. **数据源配置**：报表的数据源在 EAMS2026 中通过 `query_datasource` 字段指定（`main` / `erp`），需要根据旧系统配置调整
4. **权限迁移**：旧系统的角色 ID 需要映射到 EAMS2026 的角色 ID

### 6.3 考勤迁移专项注意事项

1. **打卡记录去重**：EAMS_4.6 可能存在重复打卡记录，迁移前执行去重脚本 `016_FixCardRecordsDuplicates.sql`
2. **员工映射配置**：考勤员工的 `employee_no` 必须与系统员工的 `employee_no` 一致，才能正确映射
3. **时间处理**：PostgreSQL 的 TIMESTAMPTZ 和 SQL Server 的 DATETIME 存在时区差异，确保统一使用同一时区

### 6.4 回滚方案

如果迁移过程中出现异常，执行回滚：

```sql
-- 1. 从备份恢复
psql -U postgres -d eams2026 < eams2026_before_migration.sql

-- 2. 或手动回滚特定迁移（需谨慎）
DROP TABLE IF EXISTS rpt_report CASCADE;
DROP TABLE IF EXISTS rpt_category CASCADE;
-- ... 手动删除迁移创建的表

-- 3. 验证回滚完成
\dt
```
