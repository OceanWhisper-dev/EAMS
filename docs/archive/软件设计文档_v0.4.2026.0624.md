### 5.5 SQL Server 2005 兼容方案

- **HWATT 数据同步**：使用 `Microsoft.Data.SqlClient` 直接连接 SQL Server 2005，连接字符串追加 `Encrypt=False;TrustServerCertificate=True;` 以禁用 TLS 加密协商
- **U8 ERP 连接**：通过 `erp_settings_datasource` 表实时读取连接字符串，支持 `System.Data.SqlClient`（Windows）和 `System.Data.Odbc` + FreeTDS（Linux）双平台兼容
- ERP 数据库命名规则：`UFDATA_{账套号}_{年份}`，支持按年份切换

## 7. 安全设计

### 7.1 JWT 认证

- **签发**：登录成功后由 `JwtService` 生成 JWT Token
- **存储**：前端 localStorage 持久化（Pinia 内存 + localStorage 双写）
- **传输**：`Authorization: Bearer <token>` 请求头
- **有效期**：1440 分钟（24 小时，可配置）
- **签名算法**：HMAC-SHA256
- **密钥要求**：长度 ≥ 32 字符，必须从 `appsettings.json` 读取，不提供硬编码回退
- **SignalR 认证**：WebSocket 无法发送 Header，改为从 query string 读取 `access_token`

### 7.2 授权策略

基于 RBAC + Policy 的细粒度权限控制：

| 策略名 | 说明 | 注册方式 |
|--------|------|---------|
| `super_admin` | 超级管理员（RoleCode 匹配） | `RequireRole("super_admin")` |
| `department` | 部门管理 | `PermissionRequirement("department")` |
| `employee` | 员工管理 | `PermissionRequirement("employee")` |
| `user` | 用户管理 | `PermissionRequirement("user")` |
| `role` | 角色管理 | `PermissionRequirement("role")` |
| `permission` | 权限管理 | `PermissionRequirement("permission")` |
| `dict` | 字典管理 | `PermissionRequirement("dict")` |
| `dashboard-config` | 仪表盘配置 | `PermissionRequirement("dashboard-config")` |
| `operation-log` | 操作日志查看 | `PermissionRequirement("operation-log")` |
| `message` | 消息管理 | `PermissionRequirement("message")` |
| `attendance` | 考勤管理 | `PermissionRequirement("attendance")` |
| `data-permission` | 数据权限配置 | `PermissionRequirement("data-permission")` |
| `erp-report` | ERP 报表管理 | `PermissionRequirement("erp-report")` |
| `erp-vouchmodify` | ERP 单据修改 | `PermissionRequirement("erp-vouchmodify")` |
| `erp-settings` | ERP 设置管理 | `PermissionRequirement("erp-settings")` |

### 7.3 数据权限

基于角色的数据可见范围控制，支持 `DataScope` 枚举：

| 数据范围 | 说明 |
|----------|------|
| `DataScope.ALL` | 所有数据 |
| `DataScope.DEPARTMENT` | 本部门及子部门数据 |

### 7.4 安全中间件链

```
ExceptionMiddleware (全局异常捕获)
  → ResponseCompression (Gzip)
  → Cors (LanAccess - 开发环境允许所有来源)
  → Authentication (JWT Bearer)
  → Authorization (Policy-based)
  → OperationLogMiddleware (写操作自动审计)
  → Controllers
```

### 7.5 前端安全措施

- **路由守卫**：`router.beforeEach` 检查 Token 有效性，过期自动清除并跳转登录页
- **权限指令**：`v-permission` 自定义指令控制按钮级元素可见性
- **请求拦截器**：Axios 拦截器自动携带 `Authorization: Bearer <token>`
- **响应拦截器**：401 响应自动触发登出流程

### 7.6 其他安全措施

- 密码使用 BCrypt 哈希存储（`BCrypt.Net-Next`）
- SQL 注入防护：Dapper 参数化查询 + 动态字段名白名单过滤（`SanitizeParamName`）
- 实体软删除：所有实体继承 `BaseEntity`，通过 `is_deleted` 标志逻辑删除

### 8.3 状态管理

**auth Store（useAuthStore）：**
- Token 管理（Pinia + localStorage 双写持久化）
- 登录/登出
- 角色（roles）和权限编码（permissions）存储
- `isAdmin` 计算属性：检查是否包含 `super_admin` 角色
- `hasPermission(code)` 方法：检查是否拥有指定权限编码
- `forceChangePassword` 标志：首次登录或管理员重置密码后强制修改

**tabs Store（useTabsStore）：**
- 多标签页管理（切换 / 关闭 / 刷新）
- sessionStorage 持久化（窗口关闭时保留）

### 8.4 请求封装

`utils/request.ts` 封装的 Axios 实例：

1. **请求拦截器**：自动携带 `Authorization: Bearer <token>`
2. **响应拦截器**：
   - 业务错误（`success === false`）→ `ElMessage.error` 弹出提示
   - HTTP 401 → 自动清除 Token 并跳转登录页（排除登录请求本身）
   - 其他网络错误 → 弹出提示

### 8.5 实时通信

使用 `@microsoft/signalr` 建立 WebSocket 连接：

- **端点**：`/hubs/import`
- **用途**：考勤数据导入进度实时推送
- **认证**：通过 `accessTokenFactory` 提供 JWT Token
- **组管理**：`JoinTaskGroup` / `LeaveTaskGroup`
- **事件**：`ProgressUpdate`（进度更新）、`Completed`（导入完成）

### 9.1 开发环境端口

| 服务 | 端口 | 说明 |

| .NET API (Kestrel) | 5106 | 监听 `0.0.0.0:5106`，局域网内可访问 |

```
浏览器 → http://<dev-machine-ip>:5173 (Vite Dev Server)
              ├── /api/* → 代理 → http://localhost:5106 (Kestrel)
              └── /hubs/* → 代理 (ws) → http://localhost:5106 (SignalR)
```
