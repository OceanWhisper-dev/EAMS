## 2. 首次部署

> 本章涵盖从零开始的完整部署流程，包括 PVE LXC 容器创建、操作系统初始化、各服务安装配置。
> 适用于首次搭建生产环境或重建环境。

### 2.1 PVE LXC 容器环境准备

#### 2.1.1 下载 Debian 12 模板

在 PVE 宿主机操作：

```bash
# 在 PVE 节点下载 Debian 12 模板（如尚未下载）
pveam update
pveam available | grep debian-12-standard

# 下载模板
pveam download local debian-12-standard_12.7-1_amd64.tar.zst

# 确认模板路径
ls -lh /var/lib/vz/template/cache/debian-12-standard_*.tar.zst
```

#### 2.1.2 创建三个 LXC 容器

```bash
# ───── lxc137 (PostgreSQL) ─────
pct create 137 /var/lib/vz/template/cache/debian-12-standard_12.7-1_amd64.tar.zst \
  --hostname eams-pg \
  --storage local-lvm \
  --memory 2048 \
  --swap 512 \
  --cores 2 \
  --net0 name=eth0,bridge=vmbr0,ip=192.168.101.37/24,gw=192.168.101.1 \
  --unprivileged 0 \
  --features keyctl=1,nesting=1

# ───── lxc131 (.NET API) ─────
pct create 131 /var/lib/vz/template/cache/debian-12-standard_12.7-1_amd64.tar.zst \
  --hostname eams-api \
  --storage local-lvm \
  --memory 2048 \
  --swap 512 \
  --cores 2 \
  --net0 name=eth0,bridge=vmbr0,ip=192.168.101.31/24,gw=192.168.101.1 \
  --unprivileged 0 \
  --features keyctl=1,nesting=1

# ───── lxc132 (Nginx) ─────
pct create 132 /var/lib/vz/template/cache/debian-12-standard_12.7-1_amd64.tar.zst \
  --hostname eams-web \
  --storage local-lvm \
  --memory 1024 \
  --swap 256 \
  --cores 1 \
  --net0 name=eth0,bridge=vmbr0,ip=192.168.101.32/24,gw=192.168.101.1 \
  --unprivileged 0 \
  --features keyctl=1,nesting=1
```

> **参数说明：**
> - `--unprivileged 0` — 使用特权容器，避免 PostgreSQL/Nginx 权限问题
> - `--features keyctl=1,nesting=1` — 启用 keyctl 和嵌套虚拟化

#### 2.1.3 启动并初始化容器

```bash
# 启动容器
pct start 137 && pct start 131 && pct start 132

# 进入每个容器执行基础初始化
# 以下命令需要在三个容器中各执行一次
pct enter 137   # 同理 131、132

# 容器内执行：
apt update && apt upgrade -y
apt install -y curl wget gnupg2 ca-certificates locales sudo

# 设置时区
timedatectl set-timezone Asia/Shanghai
# 或：ln -sf /usr/share/zoneinfo/Asia/Shanghai /etc/localtime

# 生成 locale
sed -i 's/^# *\(zh_CN.UTF-8\)/\1/' /etc/locale.gen
sed -i 's/^# *\(en_US.UTF-8\)/\1/' /etc/locale.gen
locale-gen
update-locale LANG=en_US.UTF-8

# 退出容器
exit
```

### 2.2 Debian 分角色部署

按照 **lxc137 → lxc131 → lxc132** 的顺序部署，因为后面的依赖前面的服务。

#### 2.2.1 lxc137 (192.168.101.37) — PostgreSQL 数据库

> **数据库存储路径：** `/db/pgdata`
> **数据库用户：** `eams_user`

**步骤 1：安装 PostgreSQL 18**

```bash
pct enter 137

# 添加 PostgreSQL 官方源
sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'
curl -fsSL https://www.postgresql.org/media/keys/ACCC4CF8.asc | gpg --dearmor -o /etc/apt/trusted.gpg.d/postgresql.gpg
apt update
apt install -y postgresql-18

# 停止服务，准备迁移数据目录
systemctl stop postgresql

# 创建自定义数据目录（如有专用磁盘，先挂载到 /db）
mkdir -p /db/pgdata
chown -R postgres:postgres /db/pgdata
chmod 700 /db/pgdata

# 初始化到自定义目录
su - postgres -c "initdb -D /db/pgdata -E UTF8 --locale=C"
```

**步骤 2：配置 PostgreSQL**

```bash
# 编辑 postgresql.conf（数据目录已改为 /db/pgdata）
PG_CONF="/db/pgdata/postgresql.conf"
sed -i "s/^#listen_addresses = 'localhost'/listen_addresses = '*'/" $PG_CONF

# 编辑 pg_hba.conf（允许 lxc131 连接）
PG_HBA="/db/pgdata/pg_hba.conf"
echo "# EAMS - allow API server connection" >> $PG_HBA
echo "host    eams2026    eams_user    192.168.101.31/32    scram-sha-256" >> $PG_HBA

# 修改 systemd 服务使用自定义数据目录
systemctl edit postgresql
# 在编辑器中添加：
# [Service]
# Environment=PGDATA=/db/pgdata

# 或者直接修改 systemd service 文件
sed -i 's|Environment=PGDATA=.*|Environment=PGDATA=/db/pgdata|' /lib/systemd/system/postgresql.service
systemctl daemon-reload

# 启动 PostgreSQL
systemctl start postgresql
systemctl enable postgresql
```

**步骤 3：创建数据库和用户**

```bash
su - postgres
psql -c "CREATE DATABASE eams2026 WITH ENCODING 'UTF8';"
psql -c "CREATE USER eams_user WITH PASSWORD 'your_strong_password_here';"
psql -c "GRANT ALL PRIVILEGES ON DATABASE eams2026 TO eams_user;"
psql -c "ALTER DATABASE eams2026 OWNER TO eams_user;"
exit
```

**步骤 4：导入数据库备份**

```bash
# 将发布包中的 database 目录复制到 lxc137
# 在 PVE 宿主机或 Windows 开发机上执行：
# scp -r publish/v0.4.2026.0624/database/* root@192.168.101.37:/root/

# 还原数据库（以 eams_user 身份）
pg_restore -U eams_user -d eams2026 /root/eams2026_database.dump

# 如果遇到权限问题，临时使用 postgres 用户还原后再授权
# pg_restore -U postgres -d eams2026 /root/eams2026_database.dump
# psql -U postgres -d eams2026 -c "GRANT ALL ON ALL TABLES IN SCHEMA public TO eams_user;"
# psql -U postgres -d eams2026 -c "GRANT ALL ON ALL SEQUENCES IN SCHEMA public TO eams_user;"

# 验证
psql -U eams_user -d eams2026 -c "\dt" | wc -l
# 应输出 50+ 张表（含 sys_*, attendance_*, erp_* 等模块化前缀表）
```

**步骤 5：数据库优化**

```sql
-- 连接数据库
psql -U eams_user -d eams2026

-- 收集统计信息
ANALYZE;

-- 退出
\q
```

#### 2.2.3 lxc132 (192.168.101.32) — Nginx

**步骤 1：安装 Nginx**

```bash
pct enter 132

apt install -y nginx
systemctl enable nginx
systemctl start nginx
```

**步骤 2：创建部署目录并复制前端文件**

```bash
mkdir -p /var/www/eams/web

# 从发布包复制（在 PVE 宿主机或 Windows 上）
# scp -r publish/v0.4.2026.0624/web/* root@192.168.101.32:/var/www/eams/web/
```

**步骤 3：配置 Nginx 反向代理**

```bash
cat > /etc/nginx/sites-available/eams2026 << 'EOF'
# EAMS2026 Nginx 配置
# 反向代理 /api → lxc131:5106，前端静态文件 → /var/www/eams/web

upstream eams_api {
    server 192.168.101.31:5106;
    keepalive 32;
}

server {
    listen 8266;
    server_name _;
    charset utf-8;
    client_max_body_size 10M;

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
    gzip_min_length 1024;

    # 前端静态文件
    root /var/www/eams/web;
    index index.html;

    # API 反向代理
    location /api/ {
        proxy_pass http://eams_api;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection keep-alive;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 90s;
    }

    # SignalR WebSocket（用于导入进度推送）
    location /hubs/ {
        proxy_pass http://eams_api;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 86400s;
    }

    # SPA 路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location /assets/ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    access_log /var/log/nginx/eams_access.log;
    error_log /var/log/nginx/eams_error.log;
}
EOF

# 启用站点
ln -sf /etc/nginx/sites-available/eams2026 /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default

# 测试配置
nginx -t

# 重载 Nginx
systemctl reload nginx

# 验证
ss -tlnp | grep 8266
```

**步骤 4：配置 HTTPS（生产环境推荐）**

```bash
# 方式一：Let's Encrypt 免费证书（需要公网域名）
apt install -y certbot python3-certbot-nginx
certbot --nginx -d eams.example.com

# 证书自动续期（crontab）
echo "0 3 * * * /usr/bin/certbot renew --quiet" | crontab -

# 方式二：使用自签名证书（内网环境）
mkdir -p /etc/nginx/ssl
openssl req -x509 -nodes -days 3650 -newkey rsa:2048 \
  -keyout /etc/nginx/ssl/eams.key \
  -out /etc/nginx/ssl/eams.crt \
  -subj "/CN=EAMS2026"
```

### 2.3 Windows Server 2022 部署（单机开发/测试）

适用于在 Windows Server 2022 上进行开发测试或小规模部署（所有服务安装在同一台机器上）。

```powershell

# 初始化数据目录

# 启动并设置开机自启
pg_ctl start
pg_ctl register -N PostgreSQL -D "$env:PGBASE\data"
```

```powershell
# 创建数据库
```

```powershell
# 配置 API
Copy-Item -Recurse publish/v0.4.2026.0624/api/* C:\EAMS2026\api\ -Force
```

#### 2.3.4 部署前端

```powershell
# 复制前端静态文件
```

#### 2.3.5 安装并配置 Nginx

```powershell
# 下载 Nginx
scoop install nginx

# 配置同 2.2.3 步，但将 upstream 改为 127.0.0.1
```

## 4. 配置说明

### 4.1 appsettings.json 核心配置

| 配置项 | 说明 | 生产环境要求 |
|--------|------|-------------|
| `ConnectionStrings:DefaultConnection` | 主数据库连接（PostgreSQL） | Host=实际 IP，使用强密码 |
| `ConnectionStrings:HwattConnection` | HWATT 考勤数据源（SQL Server） | 仅内网可达 |
| `Jwt:SecretKey` | JWT 签名密钥 | 32+ 位随机字符（程序启动时会校验长度） |
| `Jwt:Issuer` | JWT 签发者 | 默认 EAMS2026 |
| `Jwt:Audience` | JWT 受众 | 默认 EAMS2026 |
| `Jwt:ExpireMinutes` | Token 过期时间 | 默认 1440（24 小时） |
| `ErpSettings:AccountSet` | ERP 账套号 | 默认 008 |
| `ErpSettings:DataSourceName` | ERP 数据源名称 | 对应 `erp_settings_datasource` 表中的 name 字段 |
| `DingTalk:AppKey` | 钉钉应用 AppKey | 需从钉钉开放平台获取 |
| `DingTalk:AppSecret` | 钉钉应用 AppSecret | 需从钉钉开放平台获取 |
| `DingTalk:AttendanceGroupName` | 钉钉考勤组名称 | 默认"考勤组" |
| `DingTalk:AttendanceBatchDays` | 钉钉考勤同步批量天数 | 默认 7 天 |
| `HwattImport:BrchId` | HWATT 导入分支机构 ID | 默认 3 |
| `HwattImport:DefaultSyncStartDate` | HWATT 默认同步起始日期 | 默认 2024-01-01 |
| `HwattImport:BatchDays` | HWATT 导入批量天数 | 默认 7 天 |
| `Dashboard:RecentLogLimit` | 仪表盘最近日志条数限制 | 默认 10 条 |
| `Serilog:WriteTo:File:path` | 日志文件路径 | `logs/eams2026-.log`（按日滚动） |
| `Serilog:WriteTo:File:retainedFileCountLimit` | 日志保留天数 | 默认 30 天 |

### 4.2 环境变量覆盖

敏感配置可通过环境变量覆盖（优先级高于配置文件）：

```bash
# Linux (lxc131)
export ConnectionStrings__DefaultConnection="Host=192.168.101.37;Port=5432;Database=eams2026;Username=eams_user;Password=xxx"
export Jwt__SecretKey="production-secret-key-at-least-32-chars"
```

### 4.3 生产环境安全清单

| 检查项 | 说明 |
|--------|------|
| JWT SecretKey | 必须更换为随机字符串，至少 32 位（程序启动会校验） |
| 数据库密码 | 使用强密码，不要使用默认值 |
| CORS AllowedOrigins | 仅允许实际访问域名/IP |
| Swagger | 生产环境默认已关闭（仅在 Development 环境启用） |
| HTTPS | 建议启用 SSL/TLS |
| 防火墙 | 仅开放必要端口（8266、22） |
| 日志级别 | 生产环境可调为 `Warning` |

**自动备份脚本（在 lxc137 配置 crontab）：**

```bash
#!/bin/bash
# /usr/local/bin/eams-backup.sh
BACKUP_DIR="/var/backups/eams"

RETENTION_DAYS=30

mkdir -p $BACKUP_DIR
pg_dump -U postgres -h localhost -d eams2026 -F c -f $BACKUP_DIR/eams2026_$DATE.dump
find $BACKUP_DIR -name "eams2026_*.dump" -mtime +$RETENTION_DAYS -delete
```

```bash
# crontab -e (以 postgres 用户)
```

## 6. 故障排查

### 6.1 API 启动失败

| 原因 | 排查 | 解决 |
|------|------|------|
| 数据库连不上 | `journalctl -u eams-api \| grep "Connection refused"` | 确认 lxc137 的 PostgreSQL 运行中、防火墙放行 5432 |
| 端口被占用 | `ss -tlnp \| grep 5106` | 更换端口或终止占用进程 |
| JWT 密钥未配置 | 检查日志中 "SecretKey" 错误 | 在配置文件中设置长度 >= 32 的 SecretKey |
| .NET 版本不匹配 | `dotnet --list-runtimes` | 安装 .NET 8 Runtime |

### 6.2 前端访问白屏

| 原因 | 排查 | 解决 |
|------|------|------|
| Nginx 配置错误 | `nginx -t` | 修正配置文件语法 |
| 静态文件路径错误 | 检查 Nginx root 路径 | 确认 web 文件已复制到正确目录 |
| API 不可用 | F12 → Network 查看请求 | 确认 lxc131 API 正常运行 |
| 跨域问题 | 检查 `Access-Control-Allow-Origin` | 在 API 的 CORS 配置中添加 Nginx 地址 |

### 6.3 ERP 数据源配置

| 现象 | 排查 | 解决 |
|------|------|------|
| 报表查询提示"数据源未配置" | 检查 `erp_settings_datasource` 表 | 通过系统设置 > 数据源配置添加名称为 `erp_u871_currentYear` 的 SQL Server 数据源 |
| 连接 ERP 失败 | 检查 SQL Server 网络和防火墙 | 确认 ERP 数据库服务器可达，认证信息正确 |
