## 一、需求分析

### 1.1 业务背景

考勤管理模块是 EAMS2026 的核心业务模块之一，需要实现考勤数据的自动化采集（从 HWATT 考勤机和钉钉同步）、考勤记录管理、考勤报表统计等业务功能。

### 1.2 业务需求

| 业务需求 | 描述 | 优先级 |
|---------|------|--------|
| 考勤数据同步 | 从 HWATT 数据库和钉钉 API 同步考勤员工和打卡记录 | MUST |
| 打卡记录查询 | 按员工、日期范围筛选打卡记录 | MUST |
| 考勤异常处理 | 自动识别迟到/早退/缺卡，支持手动标记事件 | MUST |
| 考勤统计报表 | 月度出勤统计、费用计算 | SHOULD |
| 排班管理 | 支持班次方案配置和员工排班 | COULD |
| 员工映射 | 考勤员工与系统员工的手动/自动关联 | MUST |
| 费用计算 | 根据考勤数据自动计算费用 | COULD |
| 数据导入 | 支持导入遗留系统考勤数据 | SHOULD |

## 三、领域模型

### 3.1 实体关系

```
AttendanceEmployee (考勤员工)
├── Id (bigint, PK)
├── Name (varchar) - 考勤系统中的员工姓名
├── EmployeeNo (varchar) - 考勤系统中的员工编号
├── SystemEmployeeId (bigint, FK→Employee.Id) - 系统员工映射（可空）
├── Department (varchar) - 考勤系统中的部门
├── SyncSource (varchar) - 数据来源 (hwatt/dingtalk/manual)
├── Status (int) - 状态
└── 关联: 1:N → AttendanceRecord

AttendanceRecord (考勤记录)
├── Id (bigint, PK)
├── EmployeeId (bigint, FK→AttendanceEmployee.Id)
├── RecordDate (date) - 考勤日期
├── ClockIn (timestamp) - 上班打卡时间
├── ClockOut (timestamp) - 下班打卡时间
├── DayType (varchar) - 日类型 (workday/weekend/holiday)
├── Status (varchar) - 状态 (normal/late/early/absent)
├── Source (varchar) - 数据来源
├── Fee (decimal) - 费用（可空）
└── 关联: N:1 → AttendanceEmployee

AttendanceSchemeClass (班次方案)
├── Id (bigint, PK)
├── Name (varchar) - 方案名称
├── StartTime (time) - 上班时间
├── EndTime (time) - 下班时间
├── LateMinutes (int) - 迟到宽限分钟数
└── 关联: 1:N → AttendancePlanTime

AttendancePlanTime (排班时间)
├── Id (bigint, PK)
├── SchemeClassId (bigint, FK)
├── PlanDate (date) - 排班日期
├── IsRestDay (bool) - 是否休息日
└── 关联: N:1 → AttendanceSchemeClass

AttendanceDayType (日类型)
├── Id (bigint, PK)
├── Date (date) - 日期
├── Type (varchar) - 类型 (workday/weekend/holiday)
└── Description (varchar) - 描述

AttendanceHoliday (节假日)
├── Id (bigint, PK)
├── Name (varchar) - 节假日名称
├── StartDate (date)
├── EndDate (date)
└── Year (int)

AttendanceEventDeclared (事件声明)
├── Id (bigint, PK)
├── EmployeeId (bigint, FK)
├── EventDate (date)
├── EventType (varchar) - 事件类型
├── Description (varchar)
└── Status (varchar)

AttendanceFeeCalculator (费用计算)
├── Id (bigint, PK)
├── Name (varchar) - 规则名称
├── Condition (jsonb) - 计算条件
├── Formula (varchar) - 计算公式
└── Enabled (bool)

AttendanceReport (考勤报表 - 视图/查询结果)
├── EmployeeId / EmployeeName
├── Department
├── TotalDays (总天数)
├── NormalDays (正常出勤)
├── LateDays (迟到)
├── EarlyDays (早退)
├── AbsentDays (缺卡)
├── HolidayDays (节假日)
├── TotalFee (总费用)
```

### 3.2 考勤计算逻辑

```
考勤报表计算流程：
1. 获取指定日期范围内的所有考勤记录
2. 按员工分组
3. 对每个员工的每条记录判断：
   a. 该日是否为节假日/休息日 → 标记为 Holiday
   b. 是否有 ClockIn 记录 → 无 → 标记为 Absent (缺卡)
   c. ClockIn 是否晚于上班时间+宽限 → 是 → 标记为 Late (迟到)
   d. 是否有 ClockOut 记录 → 无 → 标记为 Absent
   e. ClockOut 是否早于下班时间 → 是 → 标记为 Early (早退)
   f. 以上均无 → 标记为 Normal (正常)
4. 汇总统计数据
```

## 六、数据同步设计

### 6.1 HWATT 同步

```
1. HwattImportService 通过 HwattConnectionFactory 连接 SQL Server
2. 读取 AttendanceEmployee 视图/表
3. 逐条插入/更新到 PostgreSQL attendance_employees 表
4. 读取 CardRecords 表（打卡记录）
5. 使用临时表 + MERGE 语句批量同步
6. 自动尝试根据员工姓名匹配系统员工（如无匹配则标记为未映射）
```

### 6.2 钉钉同步

```
1. DingTalkService 通过钉钉开放 API 获取 access_token
2. 调用 /topapi/attendance/list 接口获取打卡记录
3. 调用 /topapi/user/list 接口获取员工信息
4. 数据写入 PostgreSQL 对应表
5. 自动匹配系统员工
```

### 6.3 SignalR 实时进度推送

```
导入任务启动 → ImportTaskService 创建任务记录
     ↓
导入进度更新 → ImportHub.SendAsync("Progress", { percentage, message })
     ↓
前端 SignalR 接收 → 显示进度条
     ↓
导入完成 → 显示结果报告
```
