## 7. 安全设计

### 7.1 JWT 认证

- **签发**：登录成功后由 `JwtService` 生成 JWT Token
- **存储**：前端 localStorage 持久化（Pinia 内存 + localStorage 双写）
- **传输**：`Authorization: Bearer <token>` 请求头
- **有效期**：1440 分钟（24 小时，可配置）
- **签名算法**：HMAC-SHA256
- **密钥要求**：长度 ≥ 32 字符，必须从 `appsettings.json` 读取，不提供硬编码回退
- **SignalR 认证**：WebSocket 无法发送 Header，改为从 query string 读取 `access_token`

### 7.2 授权策略

```vue
  :column-resize="true"
  :column-drag="true"
  @on-sort-click="handleVTableSort"
  @on-error="handleVTableError"
/>
```

**特性：**
- 虚拟滚动 — 支持万行以上数据流畅渲染
- 列宽拖动调整（原生 `columnResize`）
- 列拖拽排序（原生 `columnDrag`）
- 列排序（`sortState` + `onSortClick` 事件）
- 自定义单元格渲染（`customRender` 回调）

### 8.2 透视表组件 — VTable PivotTable

**变更历史：** `v0.4.2026.0624` 及之前使用基于 jQuery 的 `pivottable` 库（`pivotUI()` 渲染）。`v0.4.2026.0630` 替换为 `@visactor/vue-vtable` 的 `<PivotTable>` 组件。

**关键配置：**

```vue
<PivotTable
  :columns="pivotColumnsConfig"
  :records="result.rows"
  :data-config="pivotDataConfig"
  :width="'100%'"
  :height="'100%'"
  :row-drag-order="true"
  :column-drag-order="true"
/>
```

**特性：**
- 原生透视表渲染，无需额外 CSS 补丁
- 拖拽调整行/列/指标字段
- 与后端数据格式兼容（`columns[]` + `rows[]`）
- 保存/加载的透视表参数格式适配
