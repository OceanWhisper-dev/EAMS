## 四、备份方案

### 方案 A：一键脚本备份（推荐）

使用 PowerShell 脚本 `scripts/backup.ps1` 自动执行。

**优点：** 全自动、一致性强、配置脱敏、生成清单。

### 方案 B：手动备份

按以下步骤手动操作（将 `{Version}` 替换为实际的版本字符串，例如 `0.4.2026.0624_20260624_143000`）：

```powershell
# 1. 创建版本目录
mkdir backups\0.4.2026.0624_20260624_143000

# 2. 拷贝源码（排除构建产物）
robocopy src backups\0.4.2026.0624_20260624_143000\src /E /XD bin obj node_modules dist logs /XF *.tsbuildinfo

# 3. 拷贝文档
robocopy docs backups\0.4.2026.0624_20260624_143000\docs /E

# 4. 拷贝脚本
robocopy scripts backups\0.4.2026.0624_20260624_143000\scripts /E /XD bin obj

# 5. 拷贝 IDE 配置
robocopy .trae backups\0.4.2026.0624_20260624_143000\.trae /E
robocopy .vscode backups\0.4.2026.0624_20260624_143000\.vscode /E

# 6. 配置脱敏后拷贝（手动替换密码/连接串）
```

## 五、数据字典备份

数据库备份建议同时备份到**两个位置**：

### 5.1 备份到 backups/（归档）

```powershell
& "D:\Scoop\apps\postgresql\current\bin\pg_dump.exe" ^
  -h localhost -U postgres -d eams2026 ^
  -F c -f "backups\0.4.2026.0624_20260624_143000\eams2026_database.backup"

### 5.2 同步到发布包 publish/{Version}/database/（部署用）

```powershell
# 备份脚本已支持 -UpdatePublishDump 参数自动同步
.\scripts\backup.ps1 -Version "0.4.2026.0624_20260624_143000" -UpdatePublishDump

# 或手动复制
Copy-Item "backups\0.4.2026.0624_20260624_143000\eams2026_database.backup" ^
  "publish\0.4.2026.0624_20260624_143000\database\eams2026_database.dump" -Force
```

> **为什么需要同步到发布包？**
> 发布包包含完整产物（代码 + 数据），接收方拿到发布包后可直接通过
> `pg_restore -h 目标服务器 -U postgres -d eams2026 database/eams2026_database.dump`
> 还原数据库，无需额外执行迁移脚本或单独获取备份文件。

### 5.3 发布包数据库更新流程

```
备份时（backup.ps1 -UpdatePublishDump）:
  备份数据库 → backups/{Version}/eams2026_database.backup
         ↓  （同步）
  publish/{Version}/database/eams2026_database.dump

部署时:
  pg_restore  →  生产数据库  ←  应用代码
```

## 六、备份完整性检查

备份完成后检查以下项目：

| 检查项 | 方法 |
|--------|------|
| 文件完整性 | `(Get-ChildItem -Recurse $BackupDir).Count` 确认数量正常 |
| 总大小 | `(Get-ChildItem -Recurse $BackupDir \| Measure-Object Length -Sum).Sum` |
| 关键文件 | 检查 `Program.cs`、核心 `.vue` 文件、SQL 迁移脚本是否完整 |
| 配置脱敏 | 确认 `appsettings.json` 中无明文密码/连接串 |
| 未混入排除项 | 确认 `bin/`, `obj/`, `node_modules/` 等未被包含 |

## 七、版本命名规范

### 7.1 项目版本

定义在 `.csproj` 文件中的 `<Version>` 标签，例如：
- `0.4.2026.0624`

项目版本号不携带 `v` 前缀，用于追踪代码功能演进。

### 7.2 备份目录版本

备份脚本 **自动生成**，格式为：

```
{ProjectVersion}_{YYYYMMDD}_{HHMMSS}
```

例如：`0.4.2026.0624_20260624_143000`

其中：
- `ProjectVersion` — 从 `EAMS2026.Api.csproj` 的 `<Version>` 标签自动读取
- `YYYYMMDD` — 备份执行日期
- `HHMMSS` — 备份执行时间

> 每次备份都会生成唯一的时间戳，避免因多次备份导致目录冲突。

## 八、发布包与备份的关系

发布包和备份各自承担不同角色，但通过数据库 dump 关联：

| 维度 | 发布包 `publish/{v}/` | 备份 `backups/{v}/` |
|------|----------------------|---------------------|
| **目的** | 交付给部署方使用 | 源码归档、历史追溯 |
| **内容** | API 二进制 + 前端静态文件 + 数据库 dump + 迁移脚本 + 文档 | 全量源码 + 配置（脱敏）+ 数据库 dump |
| **是否可部署** | ✅ 是，直接可用 | ❌ 需先 `dotnet build` 编译 |
| **数据库** | `database/eams2026_database.dump` | `eams2026_database.backup` |
| **数据来源** | 由 `backup.ps1 -UpdatePublishDump` 同步 | `backup.ps1` 直接生成 |

> **推荐流程：**
> 1. 开发完成后执行 `backup.ps1 -UpdatePublishDump` → 自动完成备份归档 + 发布包同步
> 2. 将 `publish/{Version}/` 目录打包分发给部署方
> 3. 部署方只需复制文件和 `pg_restore`，无需编译或执行 SQL

## 九、备份脚本使用说明

```powershell
# 默认备份，版本自动生成（基于 .csproj 的 <Version> + 当前时间戳）
.\scripts\backup.ps1
# 输出示例: backups\0.4.2026.0624_20260624_143000\

# 指定版本号（格式：{ProjectVersion}_{YYYYMMDD}_{HHMMSS}）
.\scripts\backup.ps1 -Version "0.4.2026.0624_20260624_143000"

# 跳过数据库备份
.\scripts\backup.ps1 -SkipDatabase

# 跳过配置脱敏（如本地开发环境）
.\scripts\backup.ps1 -SkipMaskConfig

# 备份并将数据库同步到发布包（publish/{Version}/database/）
.\scripts\backup.ps1 -Version "0.4.2026.0624_20260624_143000" -UpdatePublishDump

# 备份并指定发布包路径
.\scripts\backup.ps1 -Version "0.4.2026.0624_20260624_143000" -UpdatePublishDump -PublishDir "publish/0.4.2026.0624_20260624_143000"
```
